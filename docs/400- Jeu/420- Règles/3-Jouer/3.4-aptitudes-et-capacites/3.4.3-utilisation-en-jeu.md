# 3.4.3. Utilisation en jeu

**Description** : Fournir les règles d'activation (ressources, fréquence), la combinaison avec les actions habituelles, et l'articulation avec les tours de jeu.

> **Règles rapides - Utilisation**
> - **Hors combat** : Les aptitudes coûtent de la **réserve de Wakfu** (généralement 1-3 points)
> - **En combat** : Les aptitudes coûtent des **PA** (généralement 2-5 PA)
> - **Capacités** : Ne coûtent **aucune ressource** (toujours actives)
> - **Limites** : Certaines aptitudes ont des limites (par repos, par jour, etc.)
> - **Tests** : Certaines aptitudes nécessitent des tests pour fonctionner

## Contenu

- **3.4.3.1. Coûts** : PA/PM/Wakfu, slots limités, temps d'activation.
- **3.4.3.2. Fenêtre d'utilisation** : Pendant un tour, hors combat, réaction.
- **3.4.3.3. Combinaisons** : Interaction avec actions, sorts, équipements.
- **3.4.3.4. Limitations** : Usage unique, recharges, tests requis.

## 3.4.3.1. Coûts

Les aptitudes et capacités ont différents types de coûts selon leur nature.

### Points d'Action (PA)

Certaines aptitudes peuvent coûter des **PA** :
- **En combat** : Les aptitudes utilisables en combat coûtent généralement des PA
- **Coûts variables** : De 1 à 5 PA selon la puissance de l'aptitude
- **Limitation** : Vous ne pouvez utiliser que les aptitudes pour lesquelles vous avez assez de PA

### Points de Mouvement (PM)

Certaines aptitudes peuvent coûter des **PM** :
- **Déplacement** : Aptitudes liées au déplacement
- **Positionnement** : Aptitudes nécessitant un positionnement spécifique
- **Coûts variables** : Généralement 1-3 PM

### Réserve de Wakfu

**Hors combat** : La plupart des aptitudes coûtent de la **réserve de Wakfu** :
- **Coût** : Chaque utilisation de vos PA maximum consomme 1 point de réserve (1 point = 1 × PA maximum, donc 6 PA au niveau 1 ou 12 PA au maximum)
- **Limitation** : Votre réserve limite l'utilisation hors combat (ex: 4 points = 4 × 6 = 24 PA au niveau 1)
- **Récupération** : La réserve se régénère après un repos long

**En combat non significatif** : Dans les combats non significatifs, les aptitudes coûtent également de la réserve de Wakfu.

### Slots limités

Certaines aptitudes peuvent avoir des **slots limités** :
- **Par jour** : Certaines aptitudes ne peuvent être utilisées qu'un certain nombre de fois par jour
- **Par repos** : Certaines aptitudes se rechargent après un repos
- **Usage unique** : Certaines aptitudes ne peuvent être utilisées qu'une seule fois par session

### Temps d'activation

Les aptitudes peuvent nécessiter du **temps** :
- **Action** : 1 action (utilise des PA)
- **Action bonus** : Action bonus (coûte moins de PA)
- **Temps réel** : Minutes, heures pour certaines aptitudes complexes

---

## 3.4.3.2. Fenêtre d'utilisation

Les aptitudes et capacités peuvent être utilisées à différents moments.

### Pendant un tour

**En combat** : Certaines aptitudes peuvent être utilisées pendant votre tour de combat :
- **Action** : Utilise une action (coûte des PA)
- **Action bonus** : Utilise une action bonus (coûte moins de PA)
- **Limitation** : Respecte les limites de PA par tour

### Hors combat

**Principal usage** : Les aptitudes sont principalement utilisées **hors combat** :
- **Exploration** : Pendant l'exploration
- **Interaction sociale** : Pendant les interactions sociales
- **Préparation** : Avant un combat ou une rencontre

**Temps libre** : Les aptitudes peuvent être utilisées pendant votre temps libre, entre les aventures.

### Réaction

Certaines aptitudes peuvent être utilisées comme **réactions** :
- **Déclencheurs** : Se déclenchent en réponse à certains événements
- **Limitations** : Généralement limitées en fréquence
- **Coûts** : Peuvent coûter des ressources même en réaction

**Note** : Dans KrosmozJDR, il n'y a pas de système de réaction général, mais certaines aptitudes peuvent fonctionner comme des réactions selon leur description.

---

## 3.4.3.3. Combinaisons

Les aptitudes et capacités peuvent interagir avec d'autres éléments du jeu.

### Interaction avec les actions

Les aptitudes peuvent être combinées avec les actions normales :
- **Séquences** : Utiliser une aptitude puis une action normale
- **Simultanées** : Certaines aptitudes peuvent être utilisées en même temps qu'une action
- **Prérequis** : Certaines actions peuvent nécessiter une aptitude préalable

### Interaction avec les sorts

Les aptitudes peuvent interagir avec les sorts :
- **Préparation** : Utiliser une aptitude pour préparer un sort
- **Renforcement** : Certaines aptitudes renforcent les sorts
- **Alternatives** : Les aptitudes peuvent être des alternatives aux sorts

**Exemple** : Un Érudit peut utiliser "Analyse" pour mieux comprendre un sort avant de le lancer.

### Interaction avec les équipements

Les équipements peuvent améliorer les aptitudes :
- **Bonus** : Certains équipements donnent des bonus aux aptitudes
- **Réduction de coût** : Réduisent le coût des aptitudes
- **Nouvelles aptitudes** : Certains équipements confèrent de nouvelles aptitudes

---

## 3.4.3.4. Limitations

Les aptitudes et capacités ont diverses limitations.

### Usage unique

Certaines aptitudes ne peuvent être utilisées qu'**une seule fois** :
- **Par session** : Une fois par session de jeu
- **Par aventure** : Une fois par aventure
- **Par niveau** : Une fois par niveau

### Recharges

Certaines aptitudes se **rechargent** :
- **Repos court** : Se rechargent après un repos court
- **Repos long** : Se rechargent après un repos long
- **Temps** : Se rechargent après un certain temps

### Tests requis

Certaines aptitudes nécessitent des **tests** :
- **Caractéristiques** : Tests de caractéristiques appropriées
- **Compétences** : Tests de compétences
- **DD** : Degré de difficulté à atteindre

**Échec** : Si le test échoue, l'aptitude peut :
- **Ne pas fonctionner** : L'aptitude échoue
- **Fonctionner partiellement** : L'aptitude fonctionne mais avec des effets réduits
- **Coûter quand même** : Les ressources sont consommées même en cas d'échec

### Limitations contextuelles

Certaines aptitudes ont des **limitations contextuelles** :
- **Environnement** : Ne fonctionnent que dans certains environnements
- **Conditions** : Nécessitent certaines conditions pour fonctionner
- **Prérequis** : Nécessitent des prérequis (niveau, caractéristiques, etc.)

## Exemples d'utilisation

### Exemple 1 : Aptitude hors combat

**Scénario** : Le groupe explore un donjon et trouve une porte verrouillée.

**Joueur (Voleur)** : "Je vais utiliser mon aptitude Crochetage pour ouvrir cette porte."

**MJ** : "D'accord, cela coûte 1 point de réserve de Wakfu. Fais un test de Dextérité (Escamotage) DD 15."

**Résultat** : Le joueur réussit et la porte s'ouvre.

### Exemple 2 : Aptitude en combat

**Scénario** : Le groupe est en combat contre des gobelins.

**Joueur (Milicien)** : "Je vais utiliser Formation tactique pour donner un bonus à mon allié."

**MJ** : "Cela coûte 2 PA. Ton allié obtient un avantage sur son prochain jet d'attaque."

**Résultat** : L'allié attaque avec avantage et réussit.

### Exemple 3 : Capacité passive

**Scénario** : Le groupe négocie avec un marchand.

**Joueur (Négociant)** : "Je négocie le prix."

**MJ** : "Grâce à ta capacité Réduction des prix, tu obtiens automatiquement une réduction de 10%."

**Résultat** : Le prix est réduit sans test supplémentaire.

### Exemple 4 : Aptitude avec test

**Scénario** : Le groupe explore un donjon et trouve une porte verrouillée avec un mécanisme complexe.

**Joueur (Voleur niveau 5)** : "Je vais utiliser mon aptitude Crochetage pour ouvrir cette porte."

**MJ** : "D'accord, cela coûte 1 point de réserve de Wakfu. Fais un test de Dextérité (Escamotage) DD 18 (porte complexe)."

**Calcul** :
- Jet : 1d20 = 14
- Modificateur Dextérité : +3
- Bonus de maîtrise (niveau 5) : +2
- Bonus d'aptitude : +1
- **Total** : 14 + 3 + 2 + 1 = **20**

**Résultat** : Le joueur réussit (20 ≥ 18) et la porte s'ouvre. La réserve de Wakfu est consommée.

### À retenir

- **Hors combat** : Les aptitudes coûtent de la réserve de Wakfu (généralement 1-3 points)
- **En combat** : Les aptitudes coûtent des PA (généralement 2-5 PA)
- **Capacités** : Ne coûtent aucune ressource (toujours actives ou contextuelles)
- **Limites** : Certaines aptitudes ont des limites (par repos, par jour, etc.)
- **Tests** : Certaines aptitudes nécessitent des tests pour fonctionner
- **Combinaisons** : Les aptitudes peuvent être combinées avec les actions, sorts et équipements
