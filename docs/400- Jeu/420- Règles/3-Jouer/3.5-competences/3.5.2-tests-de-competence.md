# 3.5.2. Tests de compétence

**Description** : Expliquer comment effectuer un test (formulation, jet, difficulté), gérer réussites critiques/échecs critiques et tests opposés.

> **Règles rapides**
> - Formule : `1d20 + modificateur de caractéristique + maîtrise (si maîtrisée) + autres bonus`
> - Comparer le résultat au **DD** fixé par le MJ (de 5 à 30)
> - 20 naturel = réussite critique ; 1 naturel = échec critique
> - Tests opposés : les deux participants lancent en même temps, le plus haut l'emporte

## Contenu

- **3.5.2.1. Formule** : Jet d20 + modificateur + maîtrise éventuelle.
- **3.5.2.2. Classes de difficulté** : Barèmes, ajustement par le MJ.
- **3.5.2.3. Réussites/échecs critiques** : Effets narratifs, bonus/malus supplémentaires.
- **3.5.2.4. Tests opposés** : Résolution simultanée, égalités.

## 3.5.2.1. Formule

Un test de compétence suit une formule précise qui combine plusieurs éléments.

### Formule de base

**1d20 + Modificateur de caractéristique + Bonus de maîtrise (si maîtrisée) + Bonus d'expertise (si experte) + Autres modificateurs**

### Composants du test

1. **Le dé (1d20)** : Lance un dé à 20 faces
2. **Modificateur de caractéristique** : Ajoute le modificateur de la caractéristique associée à la compétence
3. **Bonus de maîtrise** : Si tu maîtrises la compétence, ajoute ton bonus de maîtrise
4. **Bonus d'expertise** : Si tu as l'expertise, ajoute un bonus de maîtrise supplémentaire (double bonus)
5. **Autres modificateurs** : Bonus ou malus temporaires (équipement, sorts, états, circonstances)

### Exemple de calcul

Un personnage niveau 10 avec :
- Agilité +4
- Maîtrise en Discrétion
- Bonus de maîtrise : +3
- Équipement donnant +1 à la Discrétion

**Test de Discrétion** : 1d20 + 4 (Agilité) + 3 (Maîtrise) + 1 (Équipement) = 1d20 + 8

Si le résultat du dé est 12, le total est 20. Ce résultat est comparé à la Classe de Difficulté (DD) déterminée par le MJ.

### Exemple avec expertise

Un personnage niveau 15 avec :
- Agilité +4
- Expertise en Discrétion
- Bonus de maîtrise : +4

**Test de Discrétion** : 1d20 + 4 (Agilité) + 8 (Expertise : 2 × 4) = 1d20 + 12

### Exemple complet

> **Contexte** : Le groupe tente d'ouvrir une porte verrouillée pendant qu'un garde patrouille.
> - Personnage : Roublard niveau 7, Dextérité +3, maîtrise +3, outils de voleur (+2).
> - Formule : `1d20 + 3 (Dextérité) + 3 (maîtrise) + 2 (outils)` = `1d20 + 8`.
> - Jet : 14 → Résultat 22.
> - DD fixé par le MJ : 18 (serrure complexe).
>
> **Résultat** : La porte s'ouvre juste avant que le garde n'arrive.

## 3.5.2.2. Classes de difficulté

La **Classe de Difficulté (DD)** représente la difficulté d'une action. Le MJ détermine le DD en fonction de la complexité de l'action.

### Barème standard

| DD | Difficulté | Exemple |
|----|------------|---------|
| **5** | Très facile | Ouvrir une porte non verrouillée |
| **10** | Facile | Escalader une corde, convaincre un ami |
| **15** | Modérée | Crocheter une serrure simple, persuader un inconnu |
| **20** | Difficile | Escalader une paroi glissante, convaincre un ennemi |
| **25** | Très difficile | Crocheter une serrure complexe, convaincre un roi |
| **30** | Presque impossible | Escalader une paroi verticale sans prise, convaincre un dieu |

> **💡 Conseil MJ**
> - Démarre tes estimations sur **DD 15** (action moyenne).
> - Ajoutez ou retirez **5 points** selon l'avantage ou le désavantage des circonstances.
> - Au-delà de **DD 25**, assure-toi que la récompense justifie la difficulté.

### Ajustement par le MJ

Le MJ peut ajuster le DD selon :
- **Les circonstances** : Conditions favorables ou défavorables
- **Le temps disponible** : Plus de temps peut réduire la difficulté
- **Les outils disponibles** : Avoir les bons outils peut réduire la difficulté
- **L'aide reçue** : L'aide d'autres personnages peut réduire la difficulté

### Conseils pour le MJ

**Évaluer rapidement** :
- **Action simple** : DD 10
- **Action normale** : DD 15
- **Action difficile** : DD 20
- **Action très difficile** : DD 25

**Ajuster selon le niveau** :
- **Personnages de bas niveau** : Utilise des DD plus bas (10-15)
- **Personnages de haut niveau** : Utilise des DD plus élevés (20-25)

**Considérer les modificateurs** :
- Si un personnage a beaucoup de bonus, augmentez légèrement le DD
- Si un personnage a des malus, réduisez légèrement le DD

## 3.5.2.3. Réussites/échecs critiques

Les réussites et échecs critiques ajoutent une dimension narrative aux tests.

### Réussite critique

Une **réussite critique** se produit quand tu obtiens un **20 naturel** (le dé montre 20) sur ton test de compétence.

**Effets** :
- **Succès garanti** : Tu réussis automatiquement, même si le DD est très élevé
- **Effets narratifs** : Le MJ peut ajouter des effets narratifs positifs
- **Bonus supplémentaires** : Tu peux obtenir des bonus supplémentaires

**Exemples narratifs** :
- **Athlétisme** : Tu escalades la paroi avec une grâce exceptionnelle, impressionnant tes compagnons
- **Persuasion** : Ton argumentation est si convaincante que la personne change complètement d'avis
- **Discrétion** : Tu te déplaces si silencieusement que même les créatures les plus alertes ne te remarquent pas

### Échec critique

Un **échec critique** se produit quand tu obtiens un **1 naturel** (le dé montre 1) sur ton test de compétence.

**Effets** :
- **Échec garanti** : Tu échoues automatiquement, même si le DD est très bas
- **Effets narratifs** : Le MJ peut ajouter des effets narratifs négatifs
- **Malus supplémentaires** : Tu peux subir des malus supplémentaires

**Exemples narratifs** :
- **Athlétisme** : Tu glisses et tombes, te blessant légèrement
- **Persuasion** : Ton argumentation est si maladroite que la personne devient hostile
- **Discrétion** : Tu trébuches bruyamment, alertant tous les gardes à proximité

### Règles optionnelles

**Réussite critique sur DD élevé** :
- Certains MJ considèrent qu'une réussite critique permet de réussir même si le total ne dépasse pas le DD
- D'autres considèrent qu'une réussite critique ajoute simplement des effets narratifs

**Échec critique sur DD bas** :
- Certains MJ considèrent qu'un échec critique fait échouer même si le total dépasse le DD
- D'autres considèrent qu'un échec critique ajoute simplement des effets narratifs

**Conseil** : Discute avec ton MJ de la façon dont les réussites et échecs critiques sont gérés dans ta campagne.

## 3.5.2.4. Tests opposés

Un **test opposé** se produit quand deux personnages ou créatures tentent d'accomplir des actions qui s'opposent directement.

### Quand utiliser un test opposé

Un test opposé est utilisé quand :
- **Deux personnages tentent la même action** : Par exemple, deux personnages tentent d'attraper le même objet
- **Un personnage tente d'empêcher l'autre** : Par exemple, un personnage tente d'ouvrir une porte qu'un autre bloque
- **Deux personnages s'opposent directement** : Par exemple, un test de Discrétion contre un test de Perception

### Résolution simultanée

Les deux personnages font leur test **simultanément** :
1. Chaque personnage lance 1d20
2. Chaque personnage ajoute ses modificateurs appropriés
3. Les résultats sont comparés

### Résultat

**Le résultat le plus élevé l'emporte** :
- Le personnage avec le résultat le plus élevé réussit son action
- Le personnage avec le résultat le plus bas échoue son action

### Égalités

Si les résultats sont **identiques** :
- **La situation reste inchangée** : Aucun des deux personnages ne progresse
- **Le statu quo est maintenu** : La situation reste la même qu'avant

**Exemples** :
- Deux personnages tentent d'attraper un anneau qui tombe : Si égalité, aucun ne l'attrape
- Un personnage tente d'ouvrir une porte qu'un autre bloque : Si égalité, la porte reste fermée
- Un test de Discrétion contre Perception : Si égalité, le personnage caché n'est pas détecté mais ne progresse pas non plus

### Exemples de tests opposés

#### Discrétion vs Perception

**Scénario** : Un voleur tente de se faufiler devant un garde.

**Résolution** :
- **Voleur** : Test de Discrétion (Agilité + Maîtrise)
- **Garde** : Test de Perception (Sagesse + Maîtrise)

**Résultat** :
- Si le voleur obtient un résultat plus élevé : Il passe inaperçu
- Si le garde obtient un résultat plus élevé : Il détecte le voleur

#### Persuasion vs Perspicacité

**Scénario** : Un personnage tente de convaincre un marchand qu'un objet vaut plus qu'il ne le pense.

**Résolution** :
- **Personnage** : Test de Persuasion (Chance + Maîtrise)
- **Marchand** : Test de Perspicacité (Sagesse + Maîtrise)

**Résultat** :
- Si le personnage obtient un résultat plus élevé : Le marchand est convaincu
- Si le marchand obtient un résultat plus élevé : Il détecte la supercherie

#### Force vs Force (Athlétisme)

**Scénario** : Deux personnages tentent de pousser une porte dans des directions opposées.

**Résolution** :
- **Personnage 1** : Test d'Athlétisme (Force + Maîtrise)
- **Personnage 2** : Test d'Athlétisme (Force + Maîtrise)

**Résultat** :
- Si le personnage 1 obtient un résultat plus élevé : Il pousse la porte dans sa direction
- Si le personnage 2 obtient un résultat plus élevé : Il pousse la porte dans sa direction

---

### À retenir

- Formule classique : `1d20 + modificateur + maîtrise/expertise + bonus`.
- Ajustez le DD par paliers de 5 selon les conditions.
- 20 naturel = réussite automatique enrichie ; 1 naturel = échec automatique narratif.
- En cas d'égalité dans un test opposé, la situation reste inchangée.
