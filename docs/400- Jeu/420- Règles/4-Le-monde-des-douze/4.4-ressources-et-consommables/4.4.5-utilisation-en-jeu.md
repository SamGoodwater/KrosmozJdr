# 4.4.5. Utilisation en jeu

**Description** : Définir les règles d'utilisation (temps, limites, effets secondaires), interaction avec les métiers et fabrication.

> **Règles rapides - Utilisation en jeu**
> - **Temps** : Action standard (PA), action bonus (moins de PA), temps libre (hors combat)
> - **Limites** : Stack d'effets (cumul possible), saturation (limite de stack), overdoses (risques)
> - **Effets secondaires** : Malus potentiels, dépendances (usage excessif)
> - **Métiers** : Qui peut fabriquer/utiliser, certifications (compétences requises)

## Contenu

- **4.4.5.1. Temps d'utilisation** : Action requise, restrictions en combat.
- **4.4.5.2. Limites** : Stack d'effets, saturation, overdoses.
- **4.4.5.3. Effets secondaires** : Malus potentiels, dépendances.
- **4.4.5.4. Interaction métiers** : Qui peut fabriquer/utiliser, certifications.

---

## 4.4.5.1. Temps d'utilisation

L'utilisation des consommables prend du temps et peut être restreinte en combat.

### Action requise

**Action standard** :
- **Temps** : Utilise une action standard (coûte des PA)
- **Consommables** : La plupart des consommables nécessitent une action standard
- **Exemples** : Potions, nourritures de base

**Action bonus** :
- **Temps** : Utilise une action bonus (coûte moins de PA)
- **Consommables** : Certains consommables peuvent être utilisés comme action bonus
- **Exemples** : Certaines potions rapides, gadgets simples

**Temps libre** :
- **Temps** : Peut être utilisé pendant le temps libre
- **Consommables** : Consommables non urgents
- **Exemples** : Nourritures, potions de buff hors combat

### Restrictions en combat

**Combat actif** :
- **Restrictions** : Certains consommables ne peuvent pas être utilisés en combat actif
- **Raisons** : Nécessitent du temps ou de la concentration
- **Exemples** : Parchemins complexes, préparations longues

**Combat non significatif** :
- **Flexibilité** : Plus de flexibilité dans l'utilisation
- **Temps** : Temps moins restrictif
- **Exemples** : Utilisation plus libre des consommables

**Urgence** :
- **Consommables rapides** : Certains consommables peuvent être utilisés rapidement
- **Coût** : Coût généralement plus élevé
- **Exemples** : Potions d'urgence, gadgets tactiques

---

## 4.4.5.2. Limites

Les consommables ont des limites pour maintenir l'équilibre.

### Stack d'effets

**Règle générale** : Les effets de même type ne se cumulent généralement pas.

**Meilleur effet** : Si plusieurs consommables donnent le même type d'effet, seul le meilleur effet s'applique.

**Effets différents** : Les effets de types différents peuvent se cumuler.

**Exemples** :
- **Deux potions de force** : Seule la meilleure s'applique
- **Potion de force + Potion de vitesse** : Les deux s'appliquent (effets différents)

### Saturation

**Limite de consommation** :
- **Par tour** : Limite de consommation par tour
- **Par jour** : Limite de consommation par jour
- **Par type** : Limite par type de consommable

**Conséquences** :
- **Inefficacité** : Consommation excessive peut être inefficace
- **Malus** : Peut causer des malus si trop consommé
- **Équilibre** : Maintient l'équilibre du jeu

### Overdoses

**Définition** : Consommation excessive de consommables peut causer des overdoses.

**Effets** :
- **Malus** : Malus aux caractéristiques ou compétences
- **États** : États négatifs (nausée, confusion, etc.)
- **Durée** : Durée variable selon la gravité

**Prévention** :
- **Modération** : Consommation modérée recommandée
- **Temps** : Espacer les consommations
- **Variété** : Varier les types de consommables

---

## 4.4.5.3. Effets secondaires

Certains consommables ont des effets secondaires.

### Malus potentiels

**Types** :
- **Malus temporaires** : Malus aux caractéristiques ou compétences
- **États négatifs** : États négatifs (fatigue, nausée, etc.)
- **Dépendances** : Risque de dépendance

**Exemples** :
- **Potion puissante** : Peut causer de la fatigue après usage
- **Potion magique** : Peut causer des effets secondaires magiques
- **Potion de force** : Peut causer de la faiblesse après expiration

### Dépendances

**Risque** :
- **Usage répété** : Usage répété peut créer une dépendance
- **Symptômes** : Symptômes de manque si arrêt
- **Traitement** : Nécessite un traitement pour guérir

**Gestion** :
- **Modération** : Modération dans l'usage
- **Alternatives** : Utiliser des alternatives
- **Traitement** : Chercher un traitement si dépendance développée

### Interactions

**Interactions entre consommables** :
- **Compatible** : Certains consommables sont compatibles
- **Incompatible** : Certains consommables sont incompatibles
- **Risques** : Interactions peuvent causer des effets imprévus

**Interactions avec sorts** :
- **Renforcement** : Certains consommables renforcent les sorts
- **Affaiblissement** : Certains consommables affaiblissent les sorts
- **Annulation** : Certains consommables annulent les sorts

---

## 4.4.5.4. Interaction métiers

Les métiers déterminent qui peut fabriquer et utiliser certains consommables.

### Qui peut fabriquer

**Métiers requis** :
- **Potions** : Alchimiste
- **Nourritures** : Boulanger, Cuisinier
- **Parchemins** : Scribe, Mage
- **Gadgets** : Bricoleur, Artisan

**Niveau requis** :
- **Règle** : Doit avoir au moins le niveau du consommable pour le fabriquer
- **Exemples** : Potion niveau 10 nécessite niveau 10 en Alchimie

**Recettes** :
- **Nécessaires** : Recettes nécessaires pour fabriquer
- **Apprentissage** : Apprentissage des recettes requis
- **Création** : Possibilité de créer de nouvelles recettes

### Qui peut utiliser

**Utilisation générale** :
- **Règle** : N'importe quelle créature peut consommer un consommable
- **Restrictions** : Peu de restrictions sur l'utilisation
- **Exceptions** : Certains consommables ont des restrictions

**Restrictions spéciales** :
- **Niveau** : Certains consommables nécessitent un niveau minimum
- **Classe** : Certains consommables sont réservés à certaines classes
- **Caractéristiques** : Certains consommables nécessitent des caractéristiques minimales

### Certifications

**Certifications professionnelles** :
- **Fonction** : Certifications pour certains types de consommables
- **Avantages** : Avantages pour les certifiés
- **Obtention** : Nécessite des tests et une formation

**Exemples** :
- **Certification alchimique** : Pour fabriquer des potions avancées
- **Certification culinaire** : Pour fabriquer des plats spéciaux
- **Certification magique** : Pour fabriquer des parchemins puissants

## Exemples pratiques

### Exemple 1 : Utilisation de potion en combat

**Scénario** : Un personnage veut utiliser une potion de soin en combat.

**Action** :
- **Temps** : Utilise une action standard (coûte des PA)
- **Effet** : Restaure des PV
- **Résultat** : PV restaurés, action utilisée

### Exemple 2 : Stack d'effets

**Scénario** : Un personnage boit deux potions de force.

**Action** :
- **Première potion** : +2 Force
- **Deuxième potion** : +2 Force (même type)
- **Résultat** : Seule la première potion s'applique (pas de cumul)

### Exemple 3 : Fabrication de consommable

**Scénario** : Un personnage veut fabriquer une potion de soin niveau 5.

**Conditions** :
- **Métier** : Doit être Alchimiste niveau 5 minimum
- **Recette** : Doit connaître la recette
- **Ressources** : Doit avoir les ressources nécessaires
- **Atelier** : Doit avoir accès à un atelier d'alchimie
