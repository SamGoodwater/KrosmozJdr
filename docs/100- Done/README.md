Une confirmation débloque l'accès ; la sécurité se réactive après 1 h d'inactivité (config `AUTH_PASSWORD_INACTIVITY_TIMEOUT`). Prop Inertia `auth.password_recently_confirmed` pour éviter de redemander le mot de passe à chaque chargement de page (ex. Scrapping).

- **Scrapping (config)** : [INVENTAIRE_JSON_ET_MIGRATION_BDD_UI.md](../50-Fonctionnalités/Scrapping/INVENTAIRE_JSON_ET_MIGRATION_BDD_UI.md) — inventaire des JSON (rôle, clés utilisées), évaluation transfert BDD + UI admin, plan de migration en 3 phases.

Résumé : [smoke-test-minimal-views-summary.md](./smoke-test-minimal-views-summary.md).
- **Export RGPD « Indisponible »** : Correction du problème où l'export restait en « pending » sans date d'expiration tant qu'aucun worker de queue ne traitait le job. Ajout de l'option `PRIVACY_EXPORT_SYNC=true` pour exécuter l'export immédiatement (sans worker), config `config/privacy.php`. Doc mise à jour ([Privacy-RGPD/README.md](../50-Fonctionnalités/Privacy-RGPD/README.md)).
- **Suppression de compte RGPD — délai de rétractation et admin** : (1) Délai de rétractation (7 jours par défaut, configurable via `PRIVACY_ERASURE_WITHDRAWAL_DAYS`) : la demande n'exécute plus le job immédiatement ; la commande `privacy:process-deletion-requests` (planifiée à 02:00) traite les demandes dont le délai est passé. (2) Bannière « Récupérer mon compte » pour les utilisateurs connectés avec une demande en cours. (3) Message explicite après déconnexion et à la tentative de connexion d'un compte supprimé. (4) Admin : boutons Restaurer et Supprimer définitivement (liste + fiche utilisateur) pour les super_admins.
- **Éléments multiples (Spell & Capability)** : Constante numérique 0-29 (Neutre, Terre, Feu, Air, Eau et combinaisons), migration Capability (element string → integer), `ElementConstants` partagé, composant `ElementDisplay` avec dégradés SCSS et icônes (`storage/app/public/images/icons/caracteristics/`). Voir [PLAN_ELEMENTS_MULTIPLES.md](../50-Fonctionnalités/Elements/PLAN_ELEMENTS_MULTIPLES.md).
- **Scrapping — corrections recherche (2026-03-09)** : (1) **Panoplies** : `$populate=items` sur l’endpoint item-sets pour inclure les équipements dans la liste ; formatter `extractItemIds` gère désormais les items sous forme d’IDs directs. (2) **Ressources** : `$populate=false` + suppression serveur de `recipesThatUse` (recettes utilisant la ressource) — ces recettes sont obtenues à l’inverse via équipements/consommables. (3) **Classes** : utilisation de `breedSpellsId` de l’API breeds (toutes les variantes) en priorité, fallback spell-levels si breedSpellsId vide ; variantes via `GET /spell-variants?breedId=X` (44 sorts base+variantes). (4) **Sorts invocations** : extraction du monstre invoqué depuis les effets spell-levels (effectId 181 = Invoquer, diceNum = ID monstre DofusDB).
- **Scrapping — pipeline import UX et fiabilité (2026-03-09)** : (1) **Progression visuelle** : fallback automatique en mode synchrone si création de job > 15 s (ex. `QUEUE_CONNECTION=sync`) ou si le job reste en `queued` > 60 s (worker non lancé) ; message explicite « mode synchrone, peut prendre plusieurs minutes » pendant l'attente. (2) **Imports multiples** : guard au clic (ne pas relancer si import en cours) ; annulation du batch précédent avant d'en lancer un nouveau. (3) **Normalisation** : la réponse du fallback synchrone est convertie au format attendu (`run_id`, `summary`, `results`, `debug`). (4) Doc `SCRAPPING_ASYNC_JOBS.md` : prérequis `queue:work` + `QUEUE_CONNECTION=database`.
- **Scrapping — option respect_auto_update (2026-03-09)** : Nouvelle option d'import `respect_auto_update` (défaut `true`). Quand activée, seules les entités déjà en BDD avec `auto_update=true` sont mises à jour par les données de l'API ; celles avec `auto_update=false` sont conservées telles quelles (non remplacées). Option exposée dans le panneau « Options & historique » du dashboard scrapping et transmise aux endpoints batch/jobs.
- **Scrapping — option skip_existing (2026-03-10)** : Nouvelle option `skip_existing` pour ne pas appeler l'API ni faire de calcul sur les entités déjà en BDD qu'on ne remplacerait pas. Accélère fortement les imports en masse.
- **Scrapping — mode de mise à jour unifié (2026-03-10)** : Un seul sélecteur `update_mode` remplace les 3 options (replace_mode, respect_auto_update, skip_existing) pour éviter les contradictions. 4 choix : ignorer si existant, mettre à jour si (brouillon/raw) ET auto_update, mettre à jour tous les auto_update, forcer les mises à jour.
- **Scrapping — fiabilité jobs (2026-03-10)** : (1) `DB::reconnect()` au début de chaque job pour éviter "MySQL server has gone away" après un job long. (2) Retries (2 tentatives, backoff 30 s) pour erreurs transitoires. (3) SaveProgress tous les 5 entités au lieu de chaque entité. (4) Doc : lancer `php artisan queue:work --timeout=660` (le worker utilise 60 s par défaut, ce qui tuait les jobs vers 1 min).
- **Scrapping — recherche par niveau monstres (2026-03-09)** : Le filtre `levelMin` / `levelMax` ne fonctionnait pas pour les monstres car l’API DofusDB stocke le niveau dans `grades.0.level` et non en champ racine. Correction : option `filters.levelPath` dans la config d’entité (ex. `grades.0.level` pour monster). Items et panoplies utilisent `level` par défaut.
