# 3.4.5. Variantes et house rules

**Description** : Proposer des options pour fusionner aptitudes/capacités, simplifier les règles ou introduire des variantes selon la table.

> **Règles rapides - Variantes et house rules**
> - **Fusion** : Option 1 (tout en aptitudes), Option 2 (tout en capacités), Option 3 (système hybride)
> - **Simplification** : Versions allégées pour débutants (aptitudes uniquement, capacités uniquement)
> - **Règles optionnelles** : Cooldowns, points dédiés, limitations supplémentaires
> - **Personnalisation** : Adaptation par univers, ton de campagne, préférences de la table

## 3.4.5.1. Fusion

Il est possible de fusionner les aptitudes et capacités pour simplifier les règles.

### Méthode de fusion

**Option 1 : Tout en aptitudes**
- **Toutes les capacités deviennent des aptitudes** : Même les capacités passives deviennent des aptitudes activables
- **Avantage** : Un seul système à gérer
- **Inconvénient** : Les capacités passives deviennent plus complexes

**Option 2 : Tout en capacités**
- **Toutes les aptitudes deviennent des capacités** : Même les aptitudes actives deviennent des capacités passives ou contextuelles
- **Avantage** : Plus simple, moins de gestion
- **Inconvénient** : Perte de flexibilité et de choix tactiques

**Option 3 : Système hybride**
- **Garder la distinction** : Mais simplifier les règles de chaque type
- **Avantage** : Garde la variété tout en simplifiant
- **Inconvénient** : Toujours deux systèmes à gérer

### Recommandation

**Pour la plupart des tables** : Garder la distinction entre aptitudes et capacités offre plus de variété et de profondeur.

**Pour les débutants** : La fusion peut simplifier l'apprentissage, mais peut réduire la profondeur du gameplay.

## 3.4.5.2. Simplification

Pour les débutants ou les tables qui préfèrent la simplicité, voici des versions allégées.

### Version allégée : Aptitudes uniquement

**Règle** : Supprimez les capacités. Toutes les capacités deviennent des aptitudes activables.

**Avantages** :
- **Un seul système** : Plus simple à comprendre
- **Choix clairs** : Les joueurs choisissent quand utiliser leurs capacités
- **Moins de passif** : Tout est actif et contrôlé

**Inconvénients** :
- **Plus de gestion** : Les joueurs doivent se souvenir d'activer leurs capacités
- **Moins d'immersion** : Les capacités passives sont plus immersives

### Version allégée : Capacités uniquement

**Règle** : Supprimez les aptitudes. Toutes les aptitudes deviennent des capacités passives ou contextuelles.

**Avantages** :
- **Très simple** : Pas de gestion active
- **Immersion** : Les capacités sont toujours actives
- **Moins de ressources** : Pas besoin de gérer les coûts

**Inconvénients** :
- **Moins de choix** : Moins de contrôle pour les joueurs
- **Moins de flexibilité** : Moins d'options tactiques

### Version allégée : Choix réduit

**Règle** : Réduisez le nombre d'aptitudes et capacités disponibles.

**Avantages** :
- **Moins de choix** : Plus facile pour les débutants
- **Plus simple** : Moins de règles à apprendre

**Inconvénients** :
- **Moins de variété** : Moins d'options de personnalisation

## 3.4.5.3. Règles optionnelles

Voici des règles optionnelles que vous pouvez ajouter selon vos préférences.

### Cooldowns

**Règle** : Ajoutez des cooldowns aux aptitudes puissantes.

**Exemple** : Une aptitude puissante ne peut être utilisée qu'une fois par combat ou une fois par jour.

**Avantages** :
- **Équilibrage** : Limite l'utilisation des aptitudes puissantes
- **Tactique** : Force les joueurs à choisir le bon moment

**Inconvénients** :
- **Complexité** : Ajoute de la complexité à la gestion

### Points dédiés

**Règle** : Créez un système de points dédiés aux aptitudes.

**Exemple** : Les joueurs ont des "Points d'Aptitude" séparés des PA.

**Avantages** :
- **Clarté** : Séparation claire entre sorts et aptitudes
- **Équilibrage** : Permet un équilibrage indépendant

**Inconvénients** :
- **Complexité** : Ajoute une ressource supplémentaire à gérer

### Limitations par niveau

**Règle** : Limitez le nombre d'aptitudes utilisables par niveau.

**Exemple** : Au niveau 3, vous pouvez utiliser 1 aptitude par jour. Au niveau 6, 2 aptitudes par jour, etc.

**Avantages** :
- **Progression** : Crée une progression claire
- **Équilibrage** : Limite l'utilisation selon le niveau

**Inconvénients** :
- **Restriction** : Peut être trop restrictif pour certains joueurs

## 3.4.5.4. Personnalisation

Les aptitudes et capacités peuvent être adaptées selon l'univers et le ton de la campagne.

### Adaptation par univers

**Tone sombre** : Dans un univers sombre, les aptitudes peuvent être plus limitées ou coûter plus cher.

**Tone léger** : Dans un univers léger, les aptitudes peuvent être plus accessibles et amusantes.

**Tone réaliste** : Dans un univers réaliste, les aptitudes peuvent être plus limitées et nécessiter plus de préparation.

### Adaptation par campagne

**Campagne combat** : Dans une campagne axée sur le combat, les aptitudes de combat peuvent être plus importantes.

**Campagne exploration** : Dans une campagne axée sur l'exploration, les aptitudes d'exploration peuvent être plus importantes.

**Campagne sociale** : Dans une campagne axée sur les interactions sociales, les aptitudes sociales peuvent être plus importantes.

### Création de nouvelles aptitudes

**Avec l'accord du MJ** : Les joueurs peuvent créer de nouvelles aptitudes personnalisées.

**Équilibrage** : Les nouvelles aptitudes doivent être équilibrées avec les aptitudes existantes.

**Thématique** : Les nouvelles aptitudes doivent être cohérentes avec l'univers et le ton de la campagne.

## Conseils pour le MJ

### Choisir les variantes

- **Style de jeu** : Choisissez des variantes qui correspondent au style de jeu de votre table
- **Expérience** : Adaptez la complexité à l'expérience de vos joueurs
- **Préférences** : Respectez les préférences de votre groupe

### Implémenter les house rules

- **Communication** : Communiquez clairement les house rules à vos joueurs
- **Cohérence** : Appliquez les house rules de manière cohérente
- **Flexibilité** : Soyez flexible et ajustez si nécessaire

### Équilibrer

- **Surveillez** : Surveillez l'impact des house rules sur l'équilibre
- **Ajustez** : Ajustez si les house rules créent des déséquilibres
- **Testez** : Testez les house rules avant de les implémenter définitivement
