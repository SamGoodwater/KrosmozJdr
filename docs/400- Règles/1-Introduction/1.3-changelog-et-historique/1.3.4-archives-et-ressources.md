## 1.3.4.3. Contributions

Le projet encourage les **contributions** de la communauté.

### Comment contribuer

1. **Signaler un bug** : Utiliser les Issues GitHub ou le Discord
2. **Proposer une amélioration** : Ouvrir une Issue avec la suggestion
3. **Créer du contenu** : Proposer des scénarios, créatures, équipements
4. **Améliorer la documentation** : Corriger ou compléter les règles

### Processus de contribution

1. **Discussion** : Discuter de la contribution sur Discord ou GitHub
2. **Proposition** : Soumettre la contribution (pull request, message, etc.)
3. **Révision** : L'équipe examine la contribution
4. **Intégration** : Si acceptée, la contribution est intégrée au projet

### Charte de modification

Les contributions doivent respecter :

- **L'esprit du jeu** : Fidélité à Dofus et équilibrage
- **La cohérence** : Compatibilité avec les règles existantes
- **La qualité** : Rédaction claire et complète

### Canaux de contribution

- **GitHub** : Pour le code et les issues
- **Discord** : Pour les discussions et retours rapides
- **Site web** : Pour les retours utilisateurs (à venir)

### Reconnaissance

Les contributeurs sont reconnus dans :

- Les notes de version
- La page des contributeurs (à venir)
- Les crédits du projet
